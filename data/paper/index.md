# `data/paper/` — paper-table data index

Each CSV in this directory backs one or more table or figure citations
in the SupersessionBench paper. All files are emitted by
`scripts/build_paper_data.py` from cached responses + verdicts; **no API
calls are required to regenerate any of them**.

To regenerate everything in this directory from scratch:

```bash
python scripts/build_paper_data.py
```

To regenerate every figure PDF from these CSVs:

```bash
python scripts/build_paper_figures.py
```

## Upstream sources

These are the inputs `build_paper_data.py` reads (all in-repo under `data/`):

- `data/dataset/realized_phase3_main_full.jsonl` — 1{,}000 gold samples (public + `_gold`)
- `data/manifests/phase3_main.json` — 350-group matched manifest
- `data/verdicts/phase3_xsystem_opus_verdicts.jsonl` — ~13{,}000 Opus 4.6 verdicts
- `data/verdicts/phase3_xsystem_opus_verdicts_{gemini25,gpt54,structured_gpt54_v3}.jsonl` —
  vendor-extension verdict shards
- `data/verdicts/phase3_xjudge_mistral_verdicts.jsonl` — Mistral cross-judge pool
- `data/verdicts/phase3_xjudge_mistral_sonnet_extract_drift.jsonl` — sonnet_extract drift gap-fill

To re-generate any of the upstream cached files (model-credentialed), see
the per-stage runner commands in `README.md`.

## Schema notes

- All `_pp` columns are signed percentage points (e.g. `+24.2`, `-0.3`).
- All `_vf` columns are percentages 0–100 (e.g. `59.4`).
- All CIs are 95% paired bootstrap (n_boot=2000 unless otherwise noted, seed=42).
- `n` columns are sample counts (paired sample_ids when applicable).
- `null` or empty cells indicate the dimension is not applicable to that row
  (for example, graphiti has 994 valid samples, not 1{,}000).
